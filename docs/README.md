# Documentation

MkDocs Material site with mkdocstrings API reference. Built HTML is written to `site/`
(not committed).

## Prerequisites

[uv](https://docs.astral.sh/uv/) manages the environment and dependencies.

## Build

```bash
uv sync --group docs
python3 scripts/build_docs.py
```

This **autogenerates** `docs/api/*.md` stubs and the API section of `mkdocs.yml` from
`src/agentTaxonomy/`, then runs MkDocs. Do not edit stub files by hand—they are
overwritten on each build.

To regenerate stubs only:

```bash
python3 scripts/generate_api_stubs.py
```

Live preview:

```bash
python3 scripts/build_docs.py --serve
# http://127.0.0.1:8000
```

## Source layout

| Path | Purpose |
|------|---------|
| `index.md` | Project overview (hand-written) |
| `getting-started.md` | Setup and first commands |
| `evaluation.md` | Scoring metrics and judges |
| `authoring.md` | Adding tasks to the catalog |
| `api/*.md` | **Autogenerated** mkdocstrings stubs (`::: agentTaxonomy.module`) |

API **content** comes from Python docstrings at build time. Stub **filenames** and
**navigation** come from `scripts/generate_api_stubs.py`.

Configuration: `mkdocs.yml` at the repository root.

## GitHub Pages

Pushes to `main` that touch docs or `src/agentTaxonomy/` trigger
[`.github/workflows/docs.yml`](../.github/workflows/docs.yml), which builds with
`scripts/build_docs.py` and deploys `site/` via GitHub Actions.

In the repository **Settings → Pages**, set **Build and deployment** source to
**GitHub Actions**. The site is published at
<https://samvritsrinath.github.io/agentTaxonomy/>.
